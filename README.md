# 🛡️ VPN Manager (vpnmgr)

Интерактивная CLI-система управления VPN-протоколами для серверов на базе **Debian 11**.  
Поддерживает VLESS+XHTTP (Xray), Hysteria2, AmneziaWG и SOCKS5.

---

## 🌟 Возможности

- **🕹️ Интерактивный TUI** — управление через меню в терминале (whiptail)
- **🔌 4 протокола** — VLESS+XHTTP (Xray), Hysteria2, AmneziaWG, SOCKS5
- **👥 Управление пользователями** — добавление, удаление, лимиты трафика, статус
- **📱 Подключение** — генерация URI и QR-кодов прямо в терминале
- **📊 Мониторинг** — статус сервисов, трафик, логи в реальном времени
- **💾 Бэкап** — ежедневные автобэкапы, восстановление из архива
- **🔁 Watchdog** — автоперезапуск упавших сервисов через cron (каждые 5 минут)
- **🔄 Обновления** — встроенный модуль обновления из репозитория
- **📜 Logrotate** — автоматическая ротация логов

## 🚀 Установка

### Шаг 1 — Клонировать репозиторий

```bash
git clone https://github.com/f2re/vpnmgr.git
cd vpnmgr
```

### Шаг 2 — Запустить установщик

```bash
sudo bash install.sh
```

Установщик выполнит:
1. Копирование файлов в `/opt/vpnmgr/`
2. Создание команды `vpnmgr` (`/usr/local/bin/vpnmgr`)
3. Установку зависимостей через `apt-get`
4. Инициализацию баз данных (`data/users.json`, `data/protocols.json`, `data/server.json`)
5. Настройку cron-задач (watchdog, ночной бэкап, проверка TLS)
6. Настройку logrotate

### Шаг 3 — Запустить

```bash
vpnmgr
```

> **Примечание:** требуются права root. Если команда не найдена — выполните `sudo vpnmgr` или проверьте симлинк `/usr/local/bin/vpnmgr`.

## ⚙️ Первоначальная настройка

После запуска `vpnmgr` откроется главное меню TUI. Рекомендуемый порядок:

### 1. Настроить сервер

Перейдите в **Протоколы** → выберите нужный протокол → **Установить / Настроить**.  
При первом запуске потребуется ввести:
- IP-адрес сервера (определяется автоматически)
- Домен и пути к TLS-сертификату (для Xray / Hysteria2)

### 2. Включить нужный протокол

Пример для VLESS+XHTTP (Xray):
- Меню: `Протоколы` → `VLESS + XHTTP (Xray)` → `Установить Xray`
- Xray устанавливается из официального источника, создаётся systemd-сервис

Пример для Hysteria2:
- Меню: `Протоколы` → `Hysteria 2` → `Установить Hysteria2`
- Поддерживаются salamander obfs, port hopping (20000–40000), masquerade URL

### 3. Добавить пользователей

Меню: `Пользователи` → `Добавить пользователя`

- Укажите имя пользователя
- При необходимости задайте лимит трафика
- Сгенерируйте URI / QR-код для передачи пользователю

### 4. Проверить статус

Меню: `Статус системы` — отображает состояние всех протоколов, IP сервера, количество пользователей.

## 📂 Структура проекта

```
vpnmgr/
├── vpnmgr.sh          # Точка входа, главное меню
├── install.sh         # Установщик
├── watchdog.sh        # Сервис-watchdog (запускается через cron)
├── lib/
│   ├── 00_core.sh     # Константы, пути, версия (v1.0.0)
│   ├── 01_utils.sh    # Вспомогательные функции
│   ├── 02_checks.sh   # Проверки системы (root, диск, терминал)
│   ├── 03_ui.sh       # Компоненты TUI (whiptail)
│   ├── 10_xray.sh     # Управление Xray (VLESS+XHTTP)
│   ├── 11_hysteria.sh # Управление Hysteria2
│   ├── 12_amnezia.sh  # Управление AmneziaWG
│   ├── 13_socks5.sh   # Управление SOCKS5-прокси
│   ├── 20_users.sh    # CRUD пользователей, лимиты трафика
│   ├── 30_monitor.sh  # Мониторинг, статус, логи
│   ├── 40_connection.sh # Генерация URI и QR-кодов
│   ├── 50_backup.sh   # Бэкап и восстановление
│   └── 51_updater.sh  # Обновление из репозитория
├── templates/         # Шаблоны конфигураций протоколов
└── data/              # Базы данных JSON (создаются при установке)
    ├── users.json     # Пользователи
    ├── protocols.json # Состояние и настройки протоколов
    └── server.json    # Параметры сервера (IP, сертификат)
```

## 🕐 Cron-задачи (настраиваются при установке)

| Расписание | Задача |
|-----------|--------|
| Каждые 5 минут | Watchdog — проверка и перезапуск сервисов |
| 03:00 ежедневно | Автобэкап (`--backup-silent`) |
| 09:00 ежедневно | Проверка срока TLS-сертификата (`--check-cert`) |

---

## 💻 CLI-режим (без TUI)

```bash
# Тихий бэкап (для cron)
vpnmgr --backup-silent

# Проверка TLS-сертификата
vpnmgr --check-cert
```

## 🐛 Устранение неполадок

- **`Запустите от имени root`** — выполните `sudo vpnmgr`
- **`whiptail: command not found`** — `sudo apt-get install whiptail`
- **Сервис не запускается** — проверьте лог: `tail -50 /opt/vpnmgr/logs/vpnmgr.log`
- **Watchdog не работает** — проверьте cron: `cat /etc/cron.d/vpnmgr`
- **Терминал слишком мал** — минимальный размер: 80×24 символа
